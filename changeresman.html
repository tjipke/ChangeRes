Short manual of ChangeRes

ChangeRes is a command line utility that allows you to change the contents of a .res (resource) file.

It can be useful in using batch building of (Delphi) projects.

Usage:

 ChangeRes Filename (-U) (-F Filename) ([VERSIONINFO]) Settings

  • Filename:    The name of the project resource file (.res extension is optional)

  • -U:    Update option file (.dof, .bdsproj, .dproj, .cbproj, .bpk) (Optional)

  • -F Filename:    Get the settings from a file (inifile format) (Optional)

  • Settings:    The things you wish to set (Optional)

From version 1.3 and higher the [VERSIONINFO] part is optional, if it is not included it presumes that you mean versioninfo.

The settings can be one of the following:

  • FileVersion=n.n.n.n(/s):
    Set the FileVersion. 'n' can be one of the following:

    • a number or a number plus a sign. To add or subtract from an existing number.
      FileVersion=+1.0.0.+1 will turn 1.2.3.4 in 2.0.0.5.
    • (new) a '*' (for the last two parts) it will use .NET automatic version numbering based on date(3 part) and time(last part).
      FileVersion=..*.* will turn 1.2.3.4 in 1.2.2046.1234 or something like that.
    • (new) a date/time format string (like yy, mm, dd, hh, nn, ss) it will set the version part depending on the datetime using the format.
      FileVersion=..yymm.ddhh will turn 1.2.3.4 in 1.2.703.1208 at 8 o'clock on March 12th 2007.
      A few things to note: a version parts maximum value = 65535 and you can use ddd for the day number in the year, so yyddd will result in 7032 for Feb 1st 2007.
    By adding /s you can tell the program to also add the line FileVersion=n.n.n.n to the strings section

  • ProductVersion=n.n.n.n(/s):
    Set the ProductVersion. See explanation of FileVersion

  • FileFlags=+/-Flags:
    Set FileFlags. FileFlags can be one of the following: VS_FF_DEBUG, VS_FF_PRERELEASE, VS_FF_PATCHED, VS_FF_PRIVATEBUILD, VS_FF_INFOINFERRED, VS_FF_SPECIALBUILD. Example: FileFlags=+VS_FF_PATCHED-VS_FF_DEBUG

  • FileDate=yyyymmddhhnnss or now or today:
    Set versioninfo's filedate. You can skip the time part and include only the date. now sets the date and time to now and today sets the date to today and leaves the time blank
    Example: Filedate=20070301 or Filedate=20070301235900 or Filedate=today

  • \StringFileInfo\\String=Value:
    Set value of a String.
    Example: \StringFileInfo\\CompanyName=Tiriss

  • -\StringFileInfo\\String:
    Remove the string.
    Example: -\StringFileInfo\\CompanyName

Examples:

 ChangeRes Project1.res -U [VERSIONINFO] FileVersion=1.2.3.4/s ProductVersion=1.2.0.0 \StringFileInfo\\CompanyName=MyCompany

 ChangeRes Project1.res FileVersion=...+1/s FileDate=20070301010203

 ChangeRes Project1 -U FileVersion=.yyddd.hhnn.ss/s ProductVersion=.yyddd.0.0/s

 ChangeRes Project1 -U -F VersionInfo.ini

Example of VersionInfo.ini:

[VERSIONINFO]
FileVersion=1.2.3.4/s
ProductVersion=1.2.0.0

 

You can only change VersionInfo parts (yet) of win32 executables and libraries with ChangeRes
